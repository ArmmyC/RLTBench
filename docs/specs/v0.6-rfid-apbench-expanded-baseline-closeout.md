# Feature Spec: v0.6 RFID-APBench Expanded Baseline Closeout

## Goal

Close the v0.6 RFID-APBench benchmark-scaling milestone after the expanded 10-task by 3-sample model baseline and analysis.

The expanded baseline analysis recommends `v0.6_expanded_baseline_closeout_spec` because the 10-task benchmark is validated, the model baseline is complete, invalid rows are classified, and benchmark assets should remain unchanged.

This closeout should produce a concise sanitized milestone report that summarizes what v0.6 accomplished, what evidence supports the milestone, what limitations remain, and what should happen next.

This is documentation and planning only. It is not task expansion, model rerun, prompt tuning, model comparison, private evaluation, fine-tuning, or training-data creation.

## Inputs

Read first:

- `reports/v0.6_rfid_apbench_expanded_baseline_analysis.md`
- `reports/v0.6_rfid_apbench_expanded_3sample_baseline.md`
- `reports/v0.6_rfid_apbench_expanded_3sample_baseline.csv`
- `reports/v0.6_rfid_apbench_task_expansion_reference_validation.md`
- `reports/v0.6_rfid_apbench_task_expansion_candidate_smoke.md`
- `reports/v0.6_rfid_apbench_post_tool_health_baseline_analysis.md`
- `reports/v0.6_rfid_apbench_3sample_baseline.md`
- `reports/v0.6_rfid_apbench_tool_health_refresh.md`
- `reports/v0.5_rfid_apbench_closeout.md`
- `benchmarks/rfid_apbench/manifest.yaml`

## Required Scope

Create a sanitized v0.6 closeout report that includes:

1. Milestone scope and non-goals.
2. Final benchmark size and public/synthetic boundary.
3. Summary of task expansion from 5 to 10 tasks.
4. Reference validation evidence for all 10 tasks.
5. `reference_copy` candidate-fixture smoke evidence for all 10 tasks.
6. Expanded 10-task by 3-sample baseline summary.
7. Expanded baseline analysis summary.
8. Known weak spots and limitations.
9. Benchmark asset decision.
10. Fine-tuning readiness decision.
11. Recommended next milestone direction.
12. Suggested tag name or release marker.

## Required Files

Create:

- `reports/v0.6_rfid_apbench_expanded_closeout.md`

Optionally update:

- `docs/specs/v0.6-rfid-apbench-expanded-baseline-closeout.md` with implementation status after the report exists

Do not create or modify model outputs, task files, prompt files, references, testbenches, extractor policy, evaluator policy, scoring formula, or benchmark data.

## Report Requirements

The closeout report must include:

- scope and non-goals
- public synthetic / no private data statement
- activity proxy caveat
- final benchmark inventory: 10 public tasks
- reference validation summary: 10/10 pass
- candidate fixture summary: 10/10 pass
- expanded baseline summary:
  - 30/30 generation completed
  - 23/30 extraction passed
  - 23/30 compile passed
  - 23/30 correctness passed
  - 22/30 synthesis passed
  - 22/30 valid scores
  - mean valid score `1.009141`
  - all-sample zero-filled mean `0.740037`
- invalid-row summary:
  - 6 empty responses
  - 1 extraction failure
  - 1 synthesis failure
- weak task summary:
  - `ap_001_idle_counter`: 0/3 valid, empty responses
  - `ap_010_retry_timeout_fsm`: 0/3 valid, empty responses
  - `ap_004_crc_serial_parallel_tradeoff`: 1 synthesis failure
  - `ap_005_fsm_controller_idle_activity`: 1 extraction failure
- statement that benchmark assets remain unchanged
- statement that fine-tuning is still not recommended solely from this evidence
- recommended next milestone direction
- suggested tag, for example `v0.6-rfid-apbench-expanded-baseline`

## Recommended Next Milestone Guidance

The closeout may recommend a next v0.7 planning direction, but must not implement it.

Acceptable next directions include:

- failure taxonomy and model-output robustness analysis
- bounded prompt-profile experiment focused on empty-response/completion reliability
- model comparison only after benchmark reports and runner paths are stable
- additional benchmark quality checks

Do not recommend fine-tuning yet unless the report clearly states it is not immediate and requires further evidence, datasets, and governance.

## Tests

Run:

```bash
python -m pytest
```

Add tests only if helper code is introduced. Do not require a live model endpoint in unit tests.

## Forbidden

Do not commit:

- private SiliconCraft RTL
- private task text
- raw prompts beyond public benchmark prompts
- raw model responses
- generated model RTL
- model outputs outside sanitized reports
- raw benchmark output directories
- VCD files
- simulator logs
- synthesis logs
- compiled simulation artifacts
- tool scratch files
- secrets or endpoint credentials
- model weights
- training datasets
- LoRA, QLoRA, or DoRA adapters
- fine-tuning scripts

## Non-Goals

Do not:

- add or modify benchmark tasks
- run another model baseline
- compare models
- tune prompts
- create training data
- fine-tune a model
- add private benchmarks
- claim measured power
- claim final silicon PPA
- change area/activity scoring formula
- change evaluator policy

## Definition of Done

Done when:

1. `reports/v0.6_rfid_apbench_expanded_closeout.md` exists.
2. It summarizes v0.6 evidence and limitations clearly.
3. It preserves public/synthetic, activity-proxy, and no-fine-tuning caveats.
4. It recommends exactly one next milestone direction or explicitly says no immediate follow-up beyond tagging.
5. It suggests a tag/release marker.
6. `python -m pytest` passes.
7. No raw/private/model-output/training artifacts are committed.
