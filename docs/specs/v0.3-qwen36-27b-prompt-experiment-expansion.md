# Feature Spec: v0.3 qwen36-27b Prompt Experiment Expansion

## 1. Goal

Expand Baseline v0.3 prompt experiments from the first two-profile VerilogEval smoke slice into a controlled, staged prompt-profile evaluation for `qwen36-27b`.

The first smoke slice has already run for VerilogEval:

- `neutral_baseline`: 3/3 passed
- `strict_rtl_only`: 3/3 passed

The next goal is to complete the smoke matrix safely before any full benchmark run.

## 2. Research Question

Can prompt profile changes improve RTL generation behavior for `qwen36-27b` while keeping model, benchmark, temperature, top-p, max tokens, evaluator, and task count controlled?

This is a prompt experiment, not a model comparison.

## 3. Prompt Profiles

The active v0.3 profiles are:

1. `neutral_baseline`
2. `strict_rtl_only`
3. `low_area_low_power`
4. `no_explanation_code_only`

The first completed slice covers only:

- `neutral_baseline`
- `strict_rtl_only`

The next slice should add:

- `low_area_low_power`
- `no_explanation_code_only`

for VerilogEval smoke only.

## 4. Staged Expansion Plan

### Stage A: Complete VerilogEval smoke

Run only the missing VerilogEval profiles:

- `low_area_low_power`
- `no_explanation_code_only`

Then regenerate the smoke summary so all four VerilogEval prompt profiles are visible in the same report.

### Stage B: Review VerilogEval smoke

Before expanding to other benchmarks, check:

- all four profiles are `finished`
- each has 3 samples
- summary counts are plausible
- no missing run metadata
- no raw outputs are committed
- prompt profile is recorded in metadata

### Stage C: RTLLM smoke

If Stage B looks clean, run RTLLM smoke for all four profiles.

### Stage D: RTL-OPT equivalence smoke

If Stage C looks clean, run RTL-OPT equivalence smoke for all four profiles.

### Stage E: Decide full-run candidate

Only after all smoke stages pass, choose one or two profiles for larger runs. Do not automatically run full benchmarks.

## 5. Required Reports

The existing smoke summary report should remain the single human-readable report for v0.3 smoke:

- `reports/v0.3_qwen36_27b_prompt_smoke_summary.md`
- `reports/v0.3_qwen36_27b_prompt_smoke_summary.csv`

The summary should show all completed and missing benchmark/profile combinations requested by the summarizer.

## 6. Commit Rules

Commit only safe reports and source/docs changes.

Allowed:

- `reports/v0.3_qwen36_27b_prompt_smoke_summary.md`
- `reports/v0.3_qwen36_27b_prompt_smoke_summary.csv`
- documentation/spec updates
- tests or helper fixes if needed

Forbidden:

- `outputs/`
- `raw_responses/`
- `extracted_rtl/`
- `logs/`
- `error_logs/`
- generated RTL code
- model responses
- prompts containing private tasks
- API keys or secrets
- caches, virtual environments, model weights

## 7. Non-goals

Do not:

- run all full benchmarks
- add new models
- change benchmark logic
- change evaluator behavior
- modify Baseline v0.1 or v0.2 artifacts
- implement LANTA serving, SSH, Slurm swapping, OpenWebUI, or LiteLLM logic
- claim PPA improvements from smoke results alone

## 8. Definition of Done for Next Codex Task

The next Codex task is done when:

1. VerilogEval smoke has finished for `low_area_low_power` and `no_explanation_code_only`.
2. The v0.3 smoke summary report includes all four VerilogEval profiles.
3. The summary CSV includes all four VerilogEval profiles.
4. `python -m pytest` passes.
5. No raw experiment outputs are committed.
6. The result summary states whether it is safe to move to RTLLM smoke.
