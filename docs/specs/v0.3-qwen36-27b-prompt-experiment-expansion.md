## 4. Required Reports

After larger validation finishes, generate:

- `reports/v0.3_qwen36_27b_prompt_larger_summary.md`
- `reports/v0.3_qwen36_27b_prompt_larger_summary.csv`

Reports should summarize only `summary.json` and `run_metadata.json` from finished larger-validation runs. Do not include raw model responses, extracted RTL, prompts, or logs.

The report should show:

- benchmark
- prompt profile
- status
- samples
- syntax/compile pass count
- functional or equivalence pass count
- failure categories
- output path

## 5. Review Questions

After the larger summary is generated, answer:

1. Which profile is stronger overall?
2. Does `strict_rtl_only` improve compile/extraction reliability?
3. Does either profile regress RTL-OPT equivalence?
4. Is there enough evidence to choose one default prompt, or should the next stage expand further?

## 6. Commit Rules

Allowed:

- larger summary reports
- documentation/spec updates only if needed
- tests or helper fixes only if needed

Forbidden:

- `outputs/`
- `raw_responses/`
- `extracted_rtl/`
- `logs/`
- `error_logs/`
- generated RTL code
- model responses
- prompts containing private tasks
- API keys or secrets
- caches, virtual environments, model weights

## 7. Non-goals

Do not:

- run full benchmarks
- run held-back profiles
- add new models
- change benchmark logic
- change evaluator behavior
- modify Baseline v0.1 or v0.2 artifacts
- implement LANTA serving, SSH, Slurm swapping, OpenWebUI, or LiteLLM logic
- claim final silicon PPA quality

## 8. Definition of Done for Next Codex Task

The next Codex task is done when:

1. Larger validation has finished for `neutral_baseline` and `strict_rtl_only`.
2. VerilogEval has 20 samples per selected profile.
3. RTLLM has 10 samples per selected profile.
4. RTL-OPT has 10 samples per selected profile.
5. Larger summary Markdown exists.
6. Larger summary CSV exists.
7. `python -m pytest` passes.
8. No raw experiment outputs are committed.
9. The final summary recommends whether to keep `neutral_baseline`, switch to `strict_rtl_only`, or expand further.
