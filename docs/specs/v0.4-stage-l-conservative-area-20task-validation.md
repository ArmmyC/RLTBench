# Feature Spec: v0.4 Stage L Conservative Area 20-Task Validation

## 1. Goal

Run a larger controlled public RTL-OPT validation for `conservative_area` after Stage K restored neutral-level correctness coverage but did not improve paired score on the five-task slice.

Stage K showed:

- `neutral_baseline`: 3 valid PPA records out of 5
- `low_area_low_power`: 2 valid PPA records out of 5
- `conservative_area`: 3 valid PPA records out of 5
- `conservative_area` matched neutral on all three paired-valid tasks
- `conservative_area` recovered `add_sub` validity versus `low_area_low_power`
- no profile produced measured power data

The next step is a larger but still bounded validation. Do not run full RTL-OPT yet.

## 2. Next Stage

### Stage L: 20-task matched `neutral_baseline` vs `conservative_area` validation

Status: next.

Run a 20-task public RTL-OPT matched validation comparing only:

- `neutral_baseline`
- `conservative_area`

Do not include `low_area_low_power` in this stage. It lost correctness coverage in Stage J and should remain held back.

## 3. Required Scope

Run only:

- benchmark: `rtlopt`
- model: `qwen36-27b`
- prompt profiles: `neutral_baseline`, `conservative_area`
- task count: 20 tasks
- samples per task: 1
- evaluator: `yosys_equiv`
- scoring: v0.4 sanitized PPA export, `area_only`, generic-cell proxy when mapped area is unavailable

Task selection:

- Use a deterministic 20-task public RTL-OPT slice.
- Include the original five Stage H/J/K tasks when possible:
  - `add_sub`
  - `adder`
  - `adder_carry`
  - `adder_select`
  - `addr_calcu`
- Record the exact task list in the report.
- Do not compare unmatched tasks.

## 4. Required Preflight

Before model calls:

1. Run `python scripts/check_yosys_techmap.py --yosys yosys`.
2. If preflight fails, stop and do not run model jobs.
3. Confirm the task list and output location.

## 5. Required Artifacts

Create sanitized artifacts:

- `artifacts/v0.4/ppa_validation20/rtlopt_neutral_baseline_ppa.jsonl`
- `artifacts/v0.4/ppa_validation20/rtlopt_conservative_area_ppa.jsonl`

Create reports:

- `reports/v0.4_ppa_validation20_neutral_summary.md`
- `reports/v0.4_ppa_validation20_neutral_summary.csv`
- `reports/v0.4_ppa_validation20_conservative_summary.md`
- `reports/v0.4_ppa_validation20_conservative_summary.csv`
- `reports/v0.4_ppa_validation20_neutral_vs_conservative.md`
- `reports/v0.4_ppa_validation20_neutral_vs_conservative.csv`

## 6. Comparison Rules

The matched comparison report must include:

- exact task list
- valid/invalid counts for both profiles
- equivalence pass counts for both profiles
- extraction/compile/synthesis pass counts for both profiles
- per-task status for both profiles
- per-task generic cell counts where available
- per-task area-only scores where valid
- paired score delta for tasks valid under both profiles
- counts of conservative-area improved, matched, and regressed tasks versus neutral
- count of tasks where conservative-area gained or lost validity versus neutral
- mean valid score for each profile
- mean all-sample score for each profile
- clear recommendation: reject, revise, keep as challenger, or promote for larger run

Do not compute deltas for invalid or unmatched tasks.

## 7. Required Caveats

Reports must state:

- `generic_cells` is a public-flow area proxy, not final silicon area
- power metrics are unavailable
- timing is not being optimized or claimed
- this is still a small public validation, not a SiliconCraft private benchmark

## 8. Success Criteria

Stage L succeeds if:

- Yosys preflight passes
- both profiles complete the same 20-task slice, or blocker is documented
- sanitized JSONL artifacts exist for both profiles if run executes
- summary Markdown/CSV exists for both profiles
- matched comparison Markdown/CSV exists
- no raw/private fields are committed
- report recommends the next action based on validity and paired score deltas

If no paired-valid tasks exist, the stage is still complete if the report clearly explains the blocker.

## 9. Commit Rules

Allowed:

- sanitized JSONL artifacts under `artifacts/v0.4/ppa_validation20/`
- Markdown/CSV reports
- docs/spec updates only if needed
- small report/export helper fixes only if needed

Forbidden:

- `outputs/`
- `raw_responses/`
- `extracted_rtl/`
- `logs/`
- `error_logs/`
- generated RTL code from model runs
- model responses
- prompts containing private tasks
- API keys or secrets
- caches, virtual environments, model weights

## 10. Non-goals

Do not:

- run full RTL-OPT
- run `low_area_low_power`
- run `strict_rtl_only`
- run VerilogEval, RTLLM, or ProtocolLLM
- run new models
- integrate SiliconCraft private benchmarks
- change v0.1/v0.2 frozen artifacts
- change v0.3 reports
- claim measured power improvement
- claim final silicon PPA quality

## 11. Definition of Done

Done when:

1. Preflight result is documented.
2. Exact 20-task list is documented.
3. Both profiles complete the matched run, or preflight/tooling blocker is clearly documented.
4. Sanitized PPA JSONL artifacts exist for executed profiles.
5. Summary Markdown/CSV reports exist.
6. Matched comparison Markdown/CSV reports exist.
7. Recommendation states whether `conservative_area` should be rejected, revised, kept as challenger, or promoted to a larger run.
8. `python -m pytest` passes.
9. No raw outputs or model responses are committed.
