# Feature Spec: v0.5 RFID-APBench Toolchain CI Validation

## Goal

Add a reproducible tool-enabled validation path for RFID-APBench references so validation does not depend on the current Codex shell having EDA tools installed.

The current reference-validation report is correct but blocked: `iverilog`, `vvp`, and `yosys` are unavailable, so all five public/synthetic tasks are marked `tool_unavailable`.

This stage should add CI or documented automation that installs/uses the required open-source tools, runs the existing reference validator, and keeps committed outputs sanitized.

## Inputs

Read first:

- `docs/specs/v0.5-rfid-apbench-tool-enabled-reference-run.md`
- `scripts/validate_rfid_apbench_references.py`
- `reports/v0.5_rfid_apbench_reference_validation.md`
- `reports/v0.5_rfid_apbench_reference_validation.csv`

## Required Scope

Implement one reproducible tool-enabled path, preferably GitHub Actions or a local script documented for Linux/LANTA, that:

1. Installs or activates `iverilog`, `vvp`, and `yosys`.
2. Runs `python -m pytest`.
3. Runs `scripts/validate_rfid_apbench_references.py` on `benchmarks/rfid_apbench`.
4. Produces sanitized Markdown/CSV reports.
5. Keeps raw VCDs, simulator binaries, logs, and synthesis scratch files under `.tmp/` or CI artifacts only.
6. Does not commit raw outputs or private/model/training artifacts.

## Preferred CI Command

The validation command should match:

```bash
python scripts/validate_rfid_apbench_references.py \
  --benchmark-root benchmarks/rfid_apbench \
  --work-dir .tmp/rfid_apbench_reference_validation \
  --output-md reports/v0.5_rfid_apbench_reference_validation.md \
  --output-csv reports/v0.5_rfid_apbench_reference_validation.csv \
  --write-reference-metrics
```

Then:

```bash
python -m pytest
```

## Acceptable Implementations

Choose one clear path:

### Option A: GitHub Actions

Add a workflow such as:

```text
.github/workflows/rfid-apbench-reference-validation.yml
```

The workflow should install Python dependencies plus Icarus Verilog and Yosys, run tests, run the reference validator, and fail if expected tool-enabled validation cannot run.

### Option B: Local/LANTA toolchain script

Add a script and docs that check tool availability and run the validator. The script must fail clearly if tools are missing and explain how to install or activate them.

## Report Policy

Do not commit regenerated reports unless the run was performed in a controlled tool-enabled environment and the outputs are sanitized.

If CI regenerates reports, prefer either:

- treating them as CI artifacts, or
- adding a documented manual commit step after review.

## Success Target

The tool-enabled run should target:

- 5 tasks loaded
- 5 compile pass
- 5 simulation pass
- 5 activity pass
- 5 synthesis pass
- 5 area pass
- 5 fully validated references

If any task fails, fix only public/synthetic task fixtures or validator bugs needed for reference validation.

## Tests

Run:

```bash
python -m pytest
```

If adding helper scripts, add tests for pure helper behavior where practical. Do not require local developer machines to have EDA tools for ordinary unit tests unless the test is clearly skipped when tools are missing.

## Forbidden

Do not commit:

- private SiliconCraft RTL
- private task text
- raw prompts
- raw model responses
- generated RTL from model runs
- model outputs
- benchmark run outputs
- VCD files
- simulator logs
- synthesis logs
- compiled simulation artifacts
- tool scratch files
- secrets
- model weights
- training datasets
- LoRA, QLoRA, or DoRA adapters
- fine-tuning scripts

## Non-Goals

Do not:

- run model evaluations
- compare models or prompt profiles
- create training data
- create a fine-tuning repository
- claim measured power
- claim final silicon PPA
- integrate private benchmarks

## Definition of Done

Done when:

1. A reproducible tool-enabled validation path exists.
2. It checks or installs Icarus Verilog and Yosys.
3. It runs pytest and the reference validator.
4. It keeps raw artifacts out of committed paths.
5. The path either validates all five references or reports tool/task blockers clearly.
6. No private/raw/model/training artifacts are committed.
