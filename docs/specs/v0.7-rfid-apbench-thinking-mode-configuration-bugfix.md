# Bugfix Spec: v0.7 RFID-APBench Thinking-Mode Configuration Propagation

## Goal

Fix the RFID-APBench model runner so it forwards the existing model preset `extra_body` into OpenAI-compatible generation requests, including `chat_template_kwargs.enable_thinking=false` for `qwen36-27b`.

The endpoint-runtime investigation recommends `thinking_mode_configuration_bugfix_spec` because the standard benchmark path loads and forwards model preset `extra_body`, but the RFID runner currently reads only endpoint/model/timeout environment values and calls the client without `extra_body`. This means the intended Qwen thinking-disable flag is absent from RFID requests.

This is a configuration propagation bugfix. It is not a benchmark change, prompt-profile experiment, model comparison, fine-tuning, private evaluation, or training-data creation.

## Required Scope

Update the RFID-APBench model-generation path so it:

1. Resolves the configured model preset from `configs/models.yaml` when available.
2. Preserves the existing endpoint environment behavior for base URL, credential, timeout, and model override.
3. Propagates the selected preset `extra_body` into `OpenAICompatibleClient.generate()`.
4. Sends `chat_template_kwargs.enable_thinking=false` for `qwen36-27b` when using the configured preset.
5. Keeps prompt profile selection, task prompts, generation settings, extractor, evaluator, scoring, and reporting unchanged except for the new extra-body plumbing.
6. Handles missing or unknown model presets safely with a clear error or documented fallback.
7. Adds mocked tests proving the extra body reaches the client payload without requiring a live endpoint.
