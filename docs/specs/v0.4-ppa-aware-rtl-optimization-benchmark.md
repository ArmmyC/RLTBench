## 3. Diagnosis and Fix Summary

The first control run had:

- samples: 5
- valid scores: 0
- invalid scores: 5
- raw run failure categories: `synthesis_failure=5`
- sanitized failure categories: `compile_failure=5`
- generated area/cells missing for 5/5

The diagnosis found that Yosys failed resolving `techmap.v` in the local OSS CAD Suite layout. The evaluator fix now resolves techmap paths using:

1. explicit env var techmap path
2. `YOSYS_DATDIR/techmap.v`
3. OSS CAD Suite-style paths near the Yosys executable
4. bare `techmap` fallback

## 5. Required Commands and Artifacts

First run the Yosys preflight:

```bash
python scripts/check_yosys_techmap.py --yosys yosys
```

If preflight fails, do not run model jobs. Report the exact failure and fix the toolchain/config first.

If preflight passes, run the same five-task neutral control. Keep raw run output under `outputs/` and do not commit it.

Then export sanitized PPA records to a new artifact:

```text
artifacts/v0.4/ppa_control/rtlopt_neutral_baseline_ppa_fixed.jsonl
```

Then summarize to:

```text
reports/v0.4_ppa_control_fixed_summary.md
reports/v0.4_ppa_control_fixed_summary.csv
```

Also add a short comparison report:

```text
reports/v0.4_ppa_control_before_after.md
```

The before/after report should compare:

- previous control summary
- fixed control summary
- compile pass count
- synth pass count
- equivalence pass count
- valid PPA records
- generated metric availability
- whether `low_area_low_power` can be unblocked

## 6. Success Criteria

Stage H succeeds if:

- Yosys techmap preflight passes
- fixed neutral control creates sanitized PPA JSONL
- summary Markdown and CSV are generated
- committed artifacts contain no raw/private fields or raw-output paths
- at least one generated sample reaches synthesis metrics, or the report clearly shows a new non-toolchain blocker

Only if neutral control produces valid correctness-gated PPA records should the next stage consider a matched `neutral_baseline` vs `low_area_low_power` retest.

## 7. Commit Rules

Allowed:

- sanitized fixed-control JSONL artifact
- fixed-control Markdown/CSV summaries
- before/after Markdown comparison report
- docs/spec updates only if needed
- small helper/test fixes only if needed

Forbidden:

- `outputs/`
- `raw_responses/`
- `extracted_rtl/`
- `logs/`
- `error_logs/`
- generated RTL code from model runs
- model responses
- prompts containing private tasks
- API keys or secrets
- caches, virtual environments, model weights

## 8. Non-goals

Do not:

- run `low_area_low_power`
- run `strict_rtl_only`
- run full benchmarks
- add new models
- commit raw logs or generated RTL
- integrate SiliconCraft private benchmarks
- change v0.1/v0.2 frozen artifacts
- change v0.3 reports
- claim power improvement unless measured power exists
- claim final silicon PPA quality

## 9. Definition of Done for Next Codex Task

The next task is done when:

1. Yosys techmap preflight result is documented in the final summary.
2. Same five-task neutral control rerun is complete, or preflight failure blocks it with a clear reason.
3. Fixed sanitized PPA JSONL exists if the rerun executes.
4. Fixed Markdown and CSV summaries exist if the rerun executes.
5. Before/after comparison report exists.
6. Report states whether valid PPA records now exist.
7. Report states whether `low_area_low_power` remains blocked or can be scheduled next.
8. `python -m pytest` passes.
9. No raw outputs are committed.
