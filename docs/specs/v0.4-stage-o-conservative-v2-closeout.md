# Feature Spec: v0.4 Stage O Conservative Area v2 Closeout

## 1. Goal

Close out the public RTL-OPT `conservative_area_v2` validation and promote it as the v0.4 public RTL-OPT challenger profile, while keeping `neutral_baseline` as the default baseline.

Stage N showed that `conservative_area_v2` is stronger than `neutral_baseline` on the 40-task public RTL-OPT validation slice by validity and all-sample score:

- `neutral_baseline`: 25 valid PPA records out of 40
- `conservative_area_v2`: 30 valid PPA records out of 40
- v2 gained validity on 5 tasks
- v2 lost validity on 0 tasks
- paired-valid tasks: 25
- v2 improved: 2
- v2 matched: 21
- v2 regressed: 2
- neutral all-sample mean: 0.727439
- v2 all-sample mean: 0.843917

The next step is not another model run. The next step is a documentation, indexing, and decision closeout so the project has a clean v0.4 milestone.

This remains a public-flow generic-cell proxy experiment. It is not final silicon PPA and has no measured power data.

## 5. Closeout Report Requirements

The Markdown closeout must include:

- final decision table for `neutral_baseline`, `low_area_low_power`, `conservative_area`, and `conservative_area_v2`
- Stage J/K/L/M/N summary timeline
- 40-task headline metrics
- validity gain/loss summary
- paired score improved/matched/regressed counts
- watchlist regressions: `adder` and `calculation`
- watchlist improvements: `comparator_16bit`, `selector`, and any other documented improved task
- held-back profiles and reasons
- clear statement that `conservative_area_v2` is the challenger, not the default baseline
- clear statement that `neutral_baseline` remains the default baseline
- caveat that `generic_cells` is a proxy, not final silicon area
- caveat that power is unavailable
- caveat that timing is not being optimized or claimed
- caveat that this is public RTL-OPT only, not SiliconCraft private validation

The CSV should contain at least:

- profile
- status: default, challenger, held_back, rejected
- best_sample_count
- valid_count
- invalid_count
- mean_valid_score
- mean_all_sample_score
- notes

## 6. Required Consistency Check

The closeout must explicitly resolve the Stage N wording ambiguity:

- Stage N used a 40-task public RTL-OPT validation slice.
- If that is the complete public RTL-OPT adapter task set, state that it effectively covers the public RTL-OPT adapter used in this repo.
- If additional public RTL-OPT tasks exist in the adapter, state what remains untested.

Do not claim coverage beyond what the repo artifacts support.

## 7. Commit Rules

Allowed:

- closeout Markdown/CSV reports
- small documentation/index updates
- this spec update if needed

Forbidden:

- new model runs
- `outputs/`
- `raw_responses/`
- `extracted_rtl/`
- `logs/`
- `error_logs/`
- generated RTL code from model runs
- model responses
- prompts containing private tasks
- API keys or secrets
- caches, virtual environments, model weights

## 8. Non-goals

Do not:

- run any benchmark
- run any prompt profile
- change prompt text
- change scoring logic
- integrate SiliconCraft private benchmarks
- claim measured power improvement
- claim final silicon PPA quality
- make `conservative_area_v2` the default model prompt

## 9. Definition of Done

Done when:

1. Closeout Markdown report exists.
2. Closeout CSV exists.
3. Decision table clearly marks `neutral_baseline` as default and `conservative_area_v2` as challenger.
4. Held-back profiles are explained.
5. Stage N 40-task coverage ambiguity is resolved honestly.
6. Any docs/index updates are small and consistent.
7. `python -m pytest` passes if code or config is changed; if only docs/reports are changed, state whether tests were skipped and why.
8. No raw outputs or model responses are committed.
